1. Purpose

This Register holds the inventory of third-party AI providers Cloudax consumes capabilities from — LLMs, ASR/TTS, foundation models, embedding services, AI tooling. It supports A.10.3 (Suppliers) and the customer-facing sub-processor disclosure required by Cloudax DPAs.

ISO 27001:2022 references: ISO 42001 reference: A.10.3 Suppliers; A.10.2 Allocating responsibilities; cross-reference to A.5.19 in the ISMS

2. Scope

This Register covers all third-party AI providers whose capabilities are consumed by Cloudax — in the platform or by personnel for Cloudax work.

3. Definitions

Definitions are per AI-POL-009 §3.

4. Roles and Responsibilities

  • AI Governance Lead: owns this Register; coordinates onboarding and review.
  • Chief Technology Officer: contributes technical assessment and integration.
  • Privacy Lead: owns sub-processor disclosure to customers.
  • Information Security Manager: contributes security assessment via PRO-021/PRO-022.

5. How to Use This Register

  • Each provider has a unique identifier (AI-3P-NNN).
  • Tier (Section 7) drives review cadence and assurance depth.
  • This document holds template + worked examples; the live register is maintained as a structured store.

6. Provider Schema

Each provider entry contains:

  • Identifier (AI-3P-NNN); provider name; capability provided.
  • Tier (Section 7).
  • Onboarding assessment date and outcome.
  • Contract reference; DPA in place (Yes / No / N/A).
  • Sub-processor (Yes / No).
  • Data handling commitments (no-training, retention, residency).
  • Hosting region(s).
  • Used by (AI system reference(s)).
  • Contract owner.
  • Last review date; next review date.
  • Exit plan reference (Tier 1 only).
  • Status (Active / Onboarding / Suspended / Exited).

9. Sub-Processor Disclosures to Customers

  • Providers marked as sub-processors are disclosed to customers per the customer DPA.
  • New sub-processors are notified to customers within the timeframe specified in the DPA (typically 30 days).
  • Customer objection process is operated per the DPA.
  • The customer-facing sub-processor list is derived from this Register, filtered to providers with Sub-Processor = Yes.

10. Review Cadence

  • Tier 1 providers: quarterly review + annual full assurance refresh.
  • Tier 2 providers: annual review.
  • Tier 3 providers: annual review.
  • Out-of-cycle review on: provider behaviour change, security incident, regulatory change.

11. Maintenance

  • This Register is updated continuously as providers are onboarded, reviewed, or exited.
  • This document structure is reviewed at least annually.
