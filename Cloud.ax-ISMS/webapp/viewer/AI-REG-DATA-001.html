1. Purpose

This Register holds the inventory of every dataset and model used in AI systems within the Cloudax AIMS scope.

ISO 27001:2022 references: ISO 42001 reference: A.4.3 Data resources; A.7.2 Data for development and enhancement of AI systems; A.7.5 Data provenance

2. Scope

This Register covers all training, evaluation, fine-tuning, reference, embedding, and prompt datasets, plus all in-house and material third-party models used in AI systems within the AIMS scope.

3. Definitions

Definitions are per AI-POL-006 §3.

4. Roles and Responsibilities

  • Chief Technology Officer: owns this Register; ensures completeness.
  • AI Builders: maintain entries for datasets and models in their domain.
  • Privacy Lead: reviews entries involving personal data.
  • AI Governance Lead: uses the Register to verify provenance and lawful basis at audit and AISIA points.

5. How to Use This Register

  • Each dataset has a unique identifier (AI-DAT-NNN).
  • Each model has a unique identifier (AI-MOD-NNN).
  • Datasets and models are cross-referenced — each model entry lists the datasets used to produce it.
  • This document holds template + worked examples; the live register is maintained as a structured store.

6. Dataset Schema

Each dataset entry contains:

  • Identifier (AI-DAT-NNN); name; description.
  • Source(s) and acquisition date; lawful basis; licence (if any).
  • Classification per POL-004.
  • Personal data (Yes / No / Pseudonymised); special category (Yes / No).
  • Quality assessment summary (date, key findings).
  • Preparation pipeline reference and version.
  • Used by (AI system reference(s) and model reference(s)).
  • Retention period and disposal plan.
  • Owner; review date.

7. Model Schema

Each model entry contains:

  • Identifier (AI-MOD-NNN); name; version.
  • Type (e.g. classifier, embedding, foundation, fine-tuned).
  • Source (in-house trained, in-house fine-tuned, third-party foundation model + provider).
  • Training datasets (cross-reference to AI-DAT-NNN entries).
  • Evaluation datasets and headline metrics.
  • Used by (AI system reference(s)).
  • Status (Production / Staging / Retired).
  • Owner; review date.

10. Relationship with Other Registers

  • AI-REG-INV-001: Each model and dataset references the AI system(s) using it.
  • AI-REG-3P-001: Third-party models and datasets reference the provider entry.
  • REG-IAR-001 (ISMS): Datasets that are also Cloudax information assets are cross-referenced.
  • REG-ROPA-001: Datasets containing personal data are reflected in ROPA entries.

11. Review Cadence

  • Production models: quarterly review.
  • Datasets in current training use: annual review.
  • Out-of-cycle review on material change.
  • Full register review annually for completeness.

12. Maintenance

  • This Register is updated as datasets and models are added, change, or are retired.
  • This document structure is reviewed at least annually.
