1. Purpose

This Policy defines how software is approved, installed, and maintained on Cloudax-managed systems.

ISO 27001:2022 references: Linked control: A.8.19 Installation of software on operational systems

2. Scope

This Policy applies to all software installed on Cloudax-managed endpoints, servers, cloud services, container images, and other operational systems within the ISMS scope.

3. Definitions

  • Approved software: software on the Cloudax approved software list, authorised for installation and use.
  • Operational system: a system used to deliver Cloudax services or to support business operations.

4. Principles

  • Only approved software may be installed on Cloudax-managed systems.
  • Software inventory is maintained and reviewed.
  • Software is sourced from official suppliers; integrity is verified where supported.
  • Installation on operational systems is controlled through automated mechanisms wherever possible.

5. Roles and Responsibilities

  • Information Security Manager: owns this Policy; approves new software for the approved software list.
  • Chief Technology Officer: approves software for production systems.
  • All personnel: install only approved software; request additions through the approved process.

6. Approved Software

  • The approved software list is maintained by the Information Security Manager.
  • New software is added only after assessment of fitness, security, licensing, and supplier risk (POL-006, POL-008).

7. Requesting New Software

  • Personnel may request new software through a documented process.
  • Requests are evaluated for business need, alternatives, security, privacy, and licensing.
  • Approved requests are added to the approved list and made available through the standard installation channel.

8. Endpoints and Standard Build

  • Cloudax-issued endpoints are deployed from a managed standard build.
  • Software installation is controlled through MDM and managed installation channels.
  • Personnel must not install unapproved software.

9. Servers, Cloud Services, and Production Systems

  • Software in production environments is deployed through automated, version-controlled mechanisms (POL-024, POL-025).
  • Manual installation is avoided; where unavoidable, it is documented and reconciled into automation.
  • Container images are built from approved base images and scanned (POL-014).

10. Open Source Software

  • OSS components introduced into Cloudax systems are subject to POL-008 IP and Software Licence Policy and the secure development controls in POL-024.

11. Browser Extensions and Plugins

  • Browser extensions can present a high risk and are subject to a managed allow-list.
  • Personnel must not install unapproved extensions on Cloudax-managed browsers.

12. AI Tools and Agents

  • AI tools, agents, and code-completion plugins are subject to specific approval, with consideration of the data submitted to the service and the supplier’s data handling (POL-006, POL-003 §12).

13. Personal Software on Cloudax Devices

  • Personal software is permitted only where it is on the approved list and consistent with reasonable personal use under POL-003.

14. Removal and End-of-Life

  • Software no longer required is removed.
  • Software at end of vendor support is removed or upgraded; exceptions are documented and risk-managed.

15. Compliance Monitoring

  • Endpoint protection and configuration management identify unapproved software.
  • Software inventory is reconciled at least annually.

16. Review and Maintenance

This Policy is reviewed at least annually and on material change.

17. Acknowledgement

All personnel are required to acknowledge this Policy on commencement, on material change, and on request.
