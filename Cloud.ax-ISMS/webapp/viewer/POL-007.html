1. Purpose

This Cloud Services Policy defines how Cloudax Ltd. uses cloud services securely, manages cloud-related risk, and maintains compliance with customer, contractual, and regulatory requirements.

ISO 27001:2022 references: Linked controls: A.5.23, A.7.1, A.7.3, A.7.6

2. Scope

This Policy applies to all use of cloud services by Cloudax, including IaaS, PaaS, SaaS, and serverless. It covers cloud services on which the Cloudax platform depends, productivity and collaboration SaaS, security and operations tooling, and any other cloud service used for Cloudax business.

3. Definitions

  • Cloud service: computing services delivered over the internet, including infrastructure, platform, and software offerings.
  • Shared responsibility model: the division of security responsibilities between Cloudax and the cloud service provider for a given service.

4. Principles

  • Cloud-native by default; on-premises deployments only by exception.
  • Established providers with mature security programmes and recognised certifications are preferred.
  • Configuration follows hardening baselines; default configurations are not assumed to be secure.
  • The shared responsibility model is documented for every cloud service in use.
  • Cloudax retains the ability to extract its data from cloud services.

5. Roles and Responsibilities

  • Chief Technology Officer: owns this Policy and the cloud architecture for the Cloudax platform.
  • Information Security Manager: approves cloud service use; reviews configuration baselines and assurance evidence.
  • Cloud service owners: named individuals responsible for each cloud service in use.

6. Cloud Service Approval

New cloud services are approved through the supplier approval process (POL-006). Approval considers:

  • Information classification of the data to be processed.
  • Provider certifications, assurance reports, and security posture.
  • Hosting region(s) and data residency.
  • Sub-processors used by the cloud provider.
  • Exit options and data portability.

7. Shared Responsibility Model

For each cloud service, the responsibilities of the provider and of Cloudax are documented, including:

  • Physical and environmental security (provider responsibility).
  • Hypervisor, host OS, and platform security (provider responsibility for IaaS/PaaS; shared for SaaS).
  • Network configuration, identity, data, and application security (Cloudax responsibility).

8. Configuration Hardening and Baselines

  • Cloud services are configured in accordance with documented hardening baselines (e.g. CIS Benchmarks, provider security best practice).
  • Configuration drift is detected through configuration management (POL-015) and remediated.
  • Public-by-default features (e.g. open storage buckets, public network access) are disabled unless required by a documented use case.

9. Identity and Access for Cloud Services

  • Access to cloud services follows POL-002 Access Control Policy.
  • Federated identity (SSO) and MFA are required for all administrative access.
  • Service-to-service authentication uses workload identities or short-lived credentials; long-lived static keys are avoided.
  • Privileged access uses separate accounts and just-in-time elevation where supported.

10. Data Residency and Sovereignty

  • Customer data is hosted in the regions agreed with the customer in the contract or DPA.
  • Cloudax maintains awareness of changes to data residency requirements and the legal environment of host countries.
  • International transfers of personal data are governed by POL-010 Privacy Policy.

11. Encryption and Key Management

  • Encryption of data in transit and at rest is implemented in accordance with POL-023 Cryptographic Controls Policy.
  • Customer-managed keys (CMK) are used where provided by customer contract or where required by data classification.

12. Logging, Monitoring, and Backup

  • Cloud service logs are forwarded to centralised logging in accordance with POL-019 Logging Policy.
  • Monitoring and alerting follow POL-020 Monitoring Policy.
  • Backups follow POL-018 Backup Policy and are tested in accordance with PLAN-DRP-001.

13. Cloud Provider Physical Security (cross-reference)

Physical security of cloud provider data centres is the responsibility of the provider. Cloudax verifies provider physical security through certifications and assurance reports, and documents this as part of supplier assurance under POL-006. Physical and environmental controls A.7.1, A.7.3 and A.7.6 are addressed by relying on cloud providers for physical perimeters and rooms/facilities, and by additional controls in POL-012 Remote Working Policy for working in secure areas at Cloudax-controlled locations.

14. Working in Cloud-Connected Spaces

Cloudax personnel access cloud services from a variety of locations — the Manchester shared workspace, home, customer sites, and travel. Working requirements for these spaces are in POL-003 Acceptable Use Policy and POL-012 Remote Working Policy.

15. Incident Coordination with Cloud Providers

  • Cloud provider incidents are monitored through provider status pages and security advisories.
  • Where a provider incident affects Cloudax services, the Cloudax incident response is invoked under PLAN-IRP-001.

16. Exit, Portability, and Termination

  • Cloud service contracts include data export and deletion provisions.
  • On termination of a cloud service, Cloudax data is exported or migrated, then verified deleted by the provider.
  • Contingency arrangements for the loss of a critical cloud service are documented in PLAN-BCP-001 / PLAN-DRP-001.

17. Review and Maintenance

This Policy is reviewed at least annually and on material change.

18. Acknowledgement

All personnel involved in cloud service use are required to acknowledge this Policy on commencement, on material change, and on request.
