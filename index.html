Settings

Flights

Data source

Stored only in this browser (localStorage); never uploaded or committed. Get a free key: subscribe to AeroDataBox on RapidAPI, then copy the X-RapidAPI-Key from the “Endpoints” tab.

Use a Cloudflare Worker proxy so the deployed board shows live data with no per-user key — the key stays server-side in the Worker. When set, the key field above is ignored.

Trains

Data source

Live trains use a public Huxley2 proxy over National Rail’s Darwin feed (best-effort uptime). For reliability you can self-host Huxley2 and point this at it. Falls back to demo data if unreachable.

Buses

Data source

Edinburgh has the built-in demo board. Other stations need an ATCO code for live data; many big ones are pre-filled, the rest show “set ATCO” until you add one below.

Live buses go through your Worker proxy (add TRANSPORTAPI_APP_ID / TRANSPORTAPI_APP_KEY secrets, set the Proxy URL above). Leave blank to use the selected station’s built-in code. A bus station has many stances, so override with the exact stance’s code — find it on bustimes.org or TransportAPI. Falls back to demo data.

EU Rail

Data source

~200 major European stations (Paris, Berlin, Amsterdam, Milano, Madrid…). UK stations are in the Trains tab. GB aside, live covers German & international long-distance via Deutsche Bahn’s open data.

Live EU trains use the public keyless DB transport.rest API over Deutsche Bahn’s HAFAS feed (best-effort uptime, CORS-enabled). You can self-host db-rest and point this at it. Falls back to demo data if unreachable.
